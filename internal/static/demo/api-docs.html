API Overview

Base Information

Base URL http://localhost:8080
API Version v1
Content Type application/json
Authentication None required

Response Formats

All endpoints support multiple response formats:

  • JSON (default): Standard JSON response with metadata
  • CSV: Comma-separated values for data export

To specify format, use either:

  • Query parameter: ?format=csv
  • Accept header: Accept: text/csv

Pagination

Search endpoints support pagination with the following parameters:

Parameter Type Default Description
limit integer 500 Number of items per page (max 500)
offset integer 0 Number of items to skip

Sorting

Most search endpoints support sorting:

Parameter Type Default Description
sort_by string varies Field to sort by
sort_order string asc Sort direction (asc/desc)

Error Handling

The API uses standard HTTP status codes:

Status Code Description
200 Success
400 Bad Request - Invalid parameters
404 Not Found - Resource doesn't exist
500 Internal Server Error

Error responses include details in JSON format:

Error Response Example

{ "success": false, "error": "Invalid player ID format", "code": 400 }

Players API Endpoints

GET /api/v1/players

Search for players with pagination and filtering.

Parameters

Parameter Type Required Description
query string No Search term for player names
limit integer No Results per page (default: 500, max: 500)
offset integer No Number of results to skip (default: 0)
sort_by string No Sort field: name, current_dwz, birth_year, club_name
sort_order string No Sort order: asc, desc (default: asc)
format string No Response format: json, csv

Example Request

cURL Example

curl -X GET "http://localhost:8080/api/v1/players?query=Mueller&limit=10&sort_by=current_dwz&sort_order=desc" \ -H "Accept: application/json"

GET /api/v1/players/{id}

Get detailed information for a specific player.

Path Parameters

Parameter Type Required Description
id string Yes Player ID in format C####-#### (e.g., C0101-1014)

Example Request

cURL Example

curl -X GET "http://localhost:8080/api/v1/players/C0101-1014" \ -H "Accept: application/json"

GET /api/v1/players/{id}/rating-history

Get the complete DWZ rating history for a player.

Path Parameters

Parameter Type Required Description
id string Yes Player ID in format C####-#### (e.g., C0101-1014)

GET /api/v1/clubs/{id}/players

Get all players belonging to a specific club.

Path Parameters

Parameter Type Required Description
id string Yes Club ID in format C#### (e.g., C0101)

This endpoint also supports the same query parameters as the players search endpoint for filtering and pagination.

Tournaments API Endpoints

GET /api/v1/tournaments

Search for tournaments with filtering and pagination.

Parameters

Parameter Type Required Description
query string No Search term for tournament names or codes
filter_by string No Filter field: year
filter_value string No Value to filter by (e.g., 2024)
sort_by string No Sort field: finishedOn (default), startedOn, name, participant_count
sort_order string No Sort order: desc (default), asc

GET /api/v1/tournaments/recent

Get recently finished tournaments.

Parameters

Parameter Type Required Description
days integer No Days to look back (default: 30)
limit integer No Maximum tournaments to return (default: 20)

GET /api/v1/tournaments/date-range

Get tournaments within a specific date range.

Parameters

Parameter Type Required Description
start_date string Yes Start date in YYYY-MM-DD format
end_date string Yes End date in YYYY-MM-DD format

Example Request

cURL Example

curl -X GET "http://localhost:8080/api/v1/tournaments/date-range?start_date=2024-01-01&end_date=2024-12-31&limit=50" \ -H "Accept: application/json"

GET /api/v1/tournaments/{id}

Get detailed information for a specific tournament.

Path Parameters

Parameter Type Required Description
id string Yes Tournament ID in format C###-###-### (e.g., C529-K00-HT1)
